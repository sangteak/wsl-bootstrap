### 핵심 돌파: "역할 인버전" 모델 (D-01 재정의)

문제의 뿌리 = *"자연스러운 편집 대상(`~/.zshrc`)이 곧 추적 파일"*. 심링크/복제본 어느 쪽이든
**우리가 `~/.zshrc`를 소유하는 한** 사용자가 각자 수정 → 충돌·오염.
해법은 소유권을 뒤집는 것:

```
[기존]
~/.zshrc ──심링크──▶ ~/.peach/dotfiles/zshrc   ← 추적 파일 = 메인(자연 편집 대상)
   └ source ~/.zshrc.local                        ← 개인(사이드카, 학습 필요)

[확정: 인버전]
~/.zshrc   ← 개인 소유 · 비추적 · 자유 편집 (자연스러운 손길 그대로)
   └ source <관리 파일>   ← 저장소 추적. 사용자는 직접 안 건드림.
```

**확정 원칙:**
> 우리는 사용자의 진입 파일(`~/.zshrc`, `~/.gitconfig` 등)을 소유하지 않는다.
> 공유 콘텐츠는 진입 파일이 `source`/`include` 하는 별도 관리 파일에 둔다.

**삼각 딜레마 해소 논리:**
- "자연 편집 대상"과 "추적 파일"을 분리하되, 자연 대상(`~/.zshrc`)을 *개인 것*으로 만들어 `.local` 학습 비용 0.
- 관리 파일은 사용자가 안 건드리니 깨끗 유지 → `pull`로 sync-down 공짜.
- 공유 = 골라서 관리 파일로 옮기는 의도적 행위 → 불필요한 개인설정이 자연 필터됨.

## 확정된 결정표

| 결정 | 확정 내용 |
|------|-----------|
| D-01 | **인버전 모델** (개인 진입 파일 + 관리 파일 source). 개인화 자유 + sync-down 공짜 동시 달성 |
| D-02 | pre-commit hook (로컬). 스캔 대상 = 관리 파일에 promote된 staged 변경분. (개인 `~/.zshrc`는 저장소에 없음) |
| D-03 | 가드 통과 시 `main` 직접 push 허용, PR은 선택 |
| D-04 | `~/.peach`는 설치·업데이트용. 기여 환경은 유연(별도 clone도 허용) |
| D-05 | 인버전으로 해소 — 각 머신 `pull` → 관리 파일 갱신, 개인 `~/.zshrc` 불간섭 |
| D-06 | 가드 탐지 패턴: 절대경로(`/home/<id>`, `/mnt/c|d/...`) / 이메일·회사도메인 / 머신명·사설IP / 토큰·API키·시크릿 |

## 페르소나 피드백 요약 (국면 2)

- **라운드 1** [비판: 🐧 Kernel Dev]: 세 모델(A 심링크 / B 복제본 / C 하이브리드)은 "누출 표면 ↔ 동기화 비용" 트레이드오프. 가드(②)가 있으면 누출 표면은 모델 무관하게 닫힘 → 모델 선택은 "sync-down을 공짜로 둘 것인가"로 축소.
- **라운드 2** [비판: 🧠 Linux Expert]: 사용자가 두 가지를 정확히 지적 — ① `.local`은 비표준 관례라 사람·LLM 모두 `~/.zshrc`를 먼저 건드림(학습 비용), ② promote-up 큐레이션은 어느 모델이든 불가피(앞선 "C는 공짜 commit" 주장은 오류, 인정). → 무게가 모델에서 가드로 이동.
- **사용자 돌파 (D-05)**: 위 두 지적을 종합해 "역할 인버전"을 제안. 페르소나 합의로 채택.

## 국면 3로 이월된 구현 매듭 (TD 검토 대상)

1. **p10k instant prompt 순서** — `~/.zshrc` 최상단 요구 vs 개인/관리 파일 source 순서.
2. **pre-commit hook 설치** — clone만으로 hook 미활성(`core.hooksPath` 또는 setup 설치 필요).
3. **초기 `~/.zshrc` 생성·재적용 멱등성** — 최초 생성하되 재실행 시 source 한 줄만 보장, 개인 내용 미덮어쓰기. 적용 대상 dotfile 범위(zshrc / gitconfig / p10k / nvim).
4. (추가) **promote 워크플로우** — 개인 `~/.zshrc`의 일부를 관리 파일로 옮기는 의도적 행위를 어떻게 매끄럽게 지원할지.
5. (추가) **가드 패턴 구현** — 정규식 오탐/미탐 균형.
