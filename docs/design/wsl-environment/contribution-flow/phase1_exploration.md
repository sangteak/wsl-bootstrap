## 본질 재구성 (인터뷰 결과)

처음엔 "로컬 변경을 *검증*하는 흐름"으로 제시됐으나, 인터뷰 중 본질이 꺾임:

> **개인/로컬 사용은 검증 불필요(자유).** 무게는 오직 **"공유 저장소에 들어가는 것"** 에만.
> 관문의 핵심 위험 = *머신 종속 절대경로·개인설정이 public 저장소에 새는 것*.
> → "검증 파이프라인"이 아니라 **기여(contribution) 흐름 + 누출 가드** 문제.

두 개의 차선이 분리되어야 함:

| 차선 | 검증 | 게이트 |
|------|------|--------|
| 🏠 개인 로컬 사용 | ❌ 불필요(자유) | 없음 |
| 🌐 저장소 반영 | 검토 중심 | 누출 차단(자동 가드 우선) + 얇은 리뷰 |

## 탐색된 요구사항 (확정/미확정 무관, 언급된 전체)

- 변경 종류: dotfiles 한 줄 수정 / ops 명령·install 모듈 추가 / 도구 버전·설치 방식 변경 / **Makefile 수정** — 전부 포함.
- 누출 차단(절대경로·개인설정)이 **최우선** 요구.
- 자동 가드가 강하면 PR 무게를 줄일 수 있다(사용자 가설).
- 여러 머신에서 사용 → 반영 후 동기화(pull·재적용) 흐름 필요.
- 리뷰어는 지정된 1인이지만 사실상 **셀프 머지**. 저장소는 **public**.
- live 환경 = `~/.peach`(심링크 원본). 부트스트랩 정상 설치 시 수정도 여기서 발생.
- **심링크 설계 자체를 재검토 대상으로 제기**: `~/.zshrc` 직접 편집이 심링크 통해 repo 원본 오염 → "원본 ↔ 복제본" 모델 고려. 핵심 의도 = "사용자는 자유롭게 수정, 저장소 반영은 경각심."
- 현재 기여 방식: 별도 clone에서 수정 후 직접 push (게이트 없음).

## 사용자가 명시적으로 제외한 항목
- 로컬 개인 변경에 대한 검증/CI — 불필요(명시).
- 무거운 다단계 승인 PR 프로세스 — 셀프 머지 환경에 부적합.

## 확정 시드 (사용자 확인 완료)

```yaml
goal: "WSL 부트스트랩 저장소의 운영 라이프사이클 정립 — 개인 환경은 자유롭게 수정하되, 저장소엔 머신 종속 절대경로·개인설정이 새지 않도록 보장하는 '개인 사용 ↔ 저장소 기여' 2-차선 흐름."
constraints:
  - "개인/로컬 사용은 검증 불필요(자유). 게이트는 오직 저장소 반영 경로에만."
  - "단일 진실 소스 = GitHub public 저장소 → PII·절대경로 노출이 곧 보안 문제."
  - "live 환경 = ~/.peach(심링크 원본), 수정도 여기서 자연스럽게 발생."
  - "여러 머신 사용 → 반영 후 동기화(pull·재적용) 흐름 포함."
  - "리뷰는 사실상 셀프(지정 리뷰어 = 본인) → 무거운 팀 프로세스 부적합."
non_goals:
  - "로컬 개인 변경에 대한 검증/CI (명시적 불필요)."
  - "무거운 다단계 승인 PR 프로세스."
success_criteria:
  - "습관대로 ~/.zshrc 등을 직접 수정해도 저장소가 오염되지 않는다."
  - "절대경로·개인설정이 커밋/푸시 전에 자동 차단·감지된다."
  - "'이 변경을 공유한다'가 의도적 행위로 분리된다."
  - "여러 머신이 충돌 없이 최신 공유 설정을 받는다."
assumptions:
  - "자동 가드가 충분히 강하면 PR 무게를 줄일 수 있다 (사용자 가설)."
  - "절대경로·이메일·머신명은 패턴으로 탐지 가능."
open_questions:
  - "dotfiles 연결 모델: 심링크 유지(+.local 강화) vs 복제본 전환 vs 하이브리드?"
  - "가드 위치: pre-commit hook(로컬) / CI(GitHub Actions) / 둘 다?"
  - "가드가 강하면 PR 완전 생략 vs 얇게 유지?"
  - "멀티 머신에서 개인 편집 ↔ 공유 업데이트 재적용 충돌 처리?"
  - "기여 출발점: live ~/.peach에서 브랜치? 별도 clone 유지?"
context: "기존 설계는 심링크 + ~/*.local 분리 패턴(머신 종속값은 .local로) 이미 존재. 그러나 사용자는 습관적으로 ~/.zshrc 직접 편집 → 심링크 통해 repo 원본 오염. 현재 기여는 별도 clone에서 직접 push(게이트 없음)."
```
